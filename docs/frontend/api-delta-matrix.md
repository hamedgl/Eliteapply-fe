# API delta matrix

Last updated: 2026-07-14

Status meanings: **integrated** is wired to Phase 1 product UI; **existing** is safely migrated in the current UI but receives richer workflow work later; **deferred** belongs to the named later phase; **admin-only** is intentionally isolated from the student application; **public-only** is restricted to a public route; **blocked** requires a contract or policy change.

This matrix includes every operation in Appendix A and every changed operation in Appendix B of the delta prompt. The current OpenAPI remains authoritative.

## Phase 3 reconciliation

The detailed table below preserves the original phase-assignment labels. Current Phase 3 implementation status is authoritative here:

| Endpoint family | Current status | Evidence |
| --- | --- | --- |
| Academic references and public referee | integrated / public-only | List, all create modes, detail/update/actions/events/downloads, header-code portal and verification routes |
| Academic interviews and audio | integrated | Durable list/detail/turn/report lifecycle plus signed recorded-audio upload and polling |
| Notifications and preferences | integrated | Cursor list, unread/read actions, shell badge, safe deep links and category preferences |
| Reminders and calendar token | integrated | Cursor list, create/update/delete/snooze and private feed create/revoke |
| Calendar ICS endpoint | public-only | Consumed by the external calendar client through the returned private URL |
| Admin Phase 3 families | admin-only | Flags, gates, operations, queue redrive, user support, moderation, reference events and reminder fallback |

Phase 3 also corrected the changed application-create workflow: programme and scholarship applications now require and submit their respective catalogue IDs instead of relying on a deep-link-only default.
