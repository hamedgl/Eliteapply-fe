# Contract gap log

Last reviewed: 2026-07-14 against `docs/api/openapi.json` (171 paths, 220 operations, 185 schemas).

| Gap                                                                        | Safe Phase 1 behavior                                                                                                                                                                                 | Follow-up                                                                                               |
| -------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------- |
| `PlanOption` has no price, currency, marketing label or feature copy.      | Public and authenticated plan selectors use only server availability, interval and token limit. The UI explicitly says the current price appears before payment; no price is fabricated.              | Add reviewed presentation metadata only when product supplies it, while keeping checkout server-driven. |
| Download responses are inconsistent across domains.                        | One defensive helper handles binary/blob, signed-URL JSON, redirects and plain-text URLs using content headers and runtime inspection. Phase 1 uses it for document, application and privacy exports. | Apply it to Writing Studio, references, calendar and other downloads in their assigned phases.          |
| Voice interviews are recorded turns, not realtime voice.                   | No Phase 1 UI claims realtime calling.                                                                                                                                                                | Keep the Phase 3 recorder turn-based.                                                                   |
| Applications expose archive but no explicit unarchive operation.           | Archive confirms that the prior stage is retained in `pre_archive_stage`; no restore control is shown.                                                                                                | Verify whether a versioned PATCH back to `pre_archive_stage` is supported before exposing restore.      |
| Notifications cannot be deleted or archived.                               | No fake controls added.                                                                                                                                                                               | Phase 3 implements only read/unread contract actions.                                                   |
| Academic and Writing Studio document lists are unpaginated arrays.         | Their query ownership is isolated; the Writing Studio list now sends `includeArchived`.                                                                                                               | Add cursor behavior only after the contract adds it.                                                    |
| Several status fields are open strings.                                    | Unknown values are rendered through a safe human-readable label function and never used to unlock features.                                                                                           | Add telemetry for unknown statuses when a privacy-reviewed analytics transport exists.                  |
| Public passcodes/reference codes are headers.                              | Referee code remains in component memory, is sent only as `X-Reference-Code`, and is cleared after submit.                                                                                            | Apply the same rule to Phase 2 share passcodes.                                                         |
| Catalogue write permissions are not self-describing.                       | Phase 1 migrates list wrappers but does not infer catalogue editing rights.                                                                                                                           | Phase 2 must use real capability/authorization responses.                                               |
| Collaborator acceptance requires authentication.                           | No Phase 1 handling was added.                                                                                                                                                                        | Phase 2 must preserve the token across login without analytics or persistent logging.                   |
| Backend deployment temporarily exposed a smaller 88-path OpenAPI snapshot. | The user-supplied 171-path repository contract is authoritative; generated types were rebuilt from it.                                                                                                | Deployment should publish the same contract to prevent environment drift.                               |
| Submission override policy is not exposed as a capability.                 | The UI never exposes override and sends `override_incomplete_requirements: false`; readiness blockers prevent submission.                                                                             | Add a policy/capability field before offering override.                                                 |
| Mutation schemas do not identify transport retry policy.                   | Mutation IDs are generated before invoking each logical action and passed as stable mutation variables; billing keeps an action-keyed registry through retries.                                       | Consider a shared transport retry classifier if automatic mutation retries are introduced.              |

No Phase 1 endpoint is blocked by a missing request or response schema.

## Phase 3 reconciliation

| Contract constraint | Implemented behavior | Remaining implication |
| --- | --- | --- |
| Interview voice is signed-upload transcription, not realtime audio or TTS. | Native `MediaRecorder` selects only a schema-supported MIME type; upload, completion and polling are separate. Text remains available. | Realtime conversation must remain absent until a new contract exists. |
| Public referee GET/submit responses are intentionally open records. | The public UI allowlists displayed fields, suppresses keys matching token/email/content/context, and keeps the code only in memory and the request header. | Backend should eventually publish a narrow public response schema. |
| Notification deep-link shape is an open data record. | The client accepts only same-origin allowlisted `/app` destinations. | New destination families require an explicit allowlist change. |
| Calendar feeds expose a private bearer URL. | The URL is shown only immediately after creation, can be copied, and can be revoked. It is never sent to analytics. | The frontend cannot recover an existing URL after navigation unless the API adds a read operation. |
| Reminder recurrence is a fixed enum. | Only none, daily, weekly and monthly are offered, using the profile timezone. | Custom recurrence rules are intentionally absent. |
| The repository has no configured client error-reporting provider. | Route recovery, scrubbed API errors and correlation-aware transport remain; no speculative SDK was added. | Add Sentry or an approved equivalent only with DSN, environment and privacy ownership. |
